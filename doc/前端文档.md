# EvalFlow Pro 前端文档（页面清单 + UI 组件 + 交互规范）

> 目标：给单人开发直接可落地的前端规格文档。
>
> 技术前提：Vue 3 + TypeScript + Vite（建议 UI：Naive UI 或 Element Plus；表格：vxe-table 或 AG Grid；富文本：Tiptap）。

---

## 1. 产品端划分与信息架构（IA）

EvalFlow Pro 前端分为两套界面：
- **用户端（User Portal）**：项目创建、资料上传、按 14 步生成、历史版本、导出、个人用量。
- **管理端（Admin Console）**：用户与账号管理、模型配置、Prompt/知识库配置、变更日志、用量统计、全量下载。

导航结构建议：
- 顶栏：产品名（EvalFlow Pro）+ 当前账号 + 退出
- 左侧：模块导航（用户端：项目/历史/用量；管理端：用户/模型/Prompt&知识库/审计/统计/下载）
- 主区：页面内容

## 2. 页面清单（必须做的所有网页）

### 2.1 公共页（未登录/通用）
1) `LoginPage` 登录
- 表单：账号/密码
- 行为：登录成功跳转项目列表

2) `RegisterPage` 注册（若开放）
- 表单：用户名、账号、密码、联系电话
- 行为：注册成功后引导登录

3) `ForgotPasswordPage`（可选）
- 若不做自助找回：显示“联系管理员”

4) `NotFoundPage` / `NoPermissionPage`

---

### 2.2 用户端页面（User Portal）

5) `ProjectListPage` 项目列表
- 功能：新建项目、搜索、排序、进入项目
- 列表字段：项目名、创建时间、更新时间、状态（进行中/已完成/已过期/已删除）
- 操作：打开、导出打包（项目全部资料）、删除（软删除）

6) `ProjectCreatePage` 新建项目（也可以做成弹窗）
- 表单：项目名称（必填）、项目类型（可选：部门/财政/自定义）、备注（可选）

7) `ProjectOverviewPage` 项目总览（进入项目后的默认页）
- 展示：项目基本信息、流程进度（14 步进度条）、近期产物、近期任务
- 入口：继续下一步、查看历史、导出

8) `ProjectWorkflowPage` 项目工作台（14 步主工作区）
- 核心：左侧步骤导航 + 中间工作区（生成/成品/历史）
- 注意：每一步页面可以复用同一“工作台骨架”，根据步骤渲染不同表单与输出组件。

9) `HistoryCenterPage` 历史记录中心（跨项目聚合，可选）
- 功能：按项目/步骤/时间筛选历史输出

10) `UsagePage` 用量与消费记录
- 列表：报告/步骤生成记录、时间、tokens、耗时、模型、项目

11) `AccountPage` 个人中心
- 展示：用户名、手机号（可修改）、密码修改

12) `DownloadsPage`（可选）
- 若导出多：提供导出任务列表（生成中/可下载/失败）

---

### 2.3 管理端页面（Admin Console）

13) `AdminDashboardPage` 管理端概览
- KPI：今日活跃用户、生成次数、tokens、失败率

14) `UserManagementPage` 用户管理
- 功能：新增账号、重置密码、禁用/启用、查看用户使用情况

15) `ModelRegistryPage` 模型配置
- 字段：平台/供应商、Base URL、API Key（脱敏显示）、模型ID、显示名、启用开关
- 行为：新增/编辑/删除/测试连通性

16) `PromptKbManagementPage` Prompt & 知识库管理
- 维度：按步骤（Step1–Step14）
- 功能：编辑 Prompt、编辑知识库条目、版本管理（草稿/发布）、回滚
- 约束：客户端不可见详细 Prompt/KB

17) `ChangeLogPage` 变更日志
- 列表：谁、何时、改了什么（Prompt/KB/模型配置）

18) `AuditLogPage` 审计日志
- 事件：登录、下载、导出、删除、配置变更

19) `UsageAnalyticsPage` 用量统计
- 维度：用户/项目/步骤/模型
- 指标：tokens、耗时、成功率、平均时长

20) `GlobalDownloadPage` 全量下载
- 管理员按用户/按项目导出 zip

## 3. 用户端“14 步”页面需求（逐步 UI 需求清单）

> 统一原则：每一步都拆成三块视图：
> 1) **生成区（可多模型对比）**
> 2) **成品区（最终版本，主界面只保留这个）**
> 3) **历史区（该步骤历史输出列表/对比/恢复）**

### Step1 生成资料清单
- 输入区：资料上传（word/excel/pdf/图片），支持多文件
- 生成区：单模型生成 + 多模型对比（Tab/分栏）
- 成品区：可编辑（富文本或 Markdown），支持“一键导入成品”
- 导出：Word/PDF（经典/自定义排版入口）

### Step2 生成有效项目资料（核心内容）
- 上传区：分两个上传框
  - 图片/PDF
  - Word/Excel
- 校验区：展示抽取文本/识别结果，提供“人工确认勾选/修订”
- 输出区：按分类展示（资金/预算/制度/实施…可扩展）
- 溯源：引用卡片（来源文件、页码、原文片段）可展开
- 成品区：可编辑 + 保存

### Step3 生成指标体系
- 输入：选择指标体系类型（项目支出/部门整体/专项债/自定义…）
- 编辑器：树形 + 表格双视图
  - 树：一级/二级/三级
  - 表：支持批量编辑“指标名/解释”
- 支持导入：上传 Excel/JSON（后期），或模板库一键导入
- 逐项生成：二级 → 三级（每个三级允许对话优化）

### Step4 生成分值
- 两种模式：自动赋分 / 人工赋分
- 赋分 UI：
  - 表格列：指标层级、名称、分值
  - 支持批量调整
- 校验提示：
  - 合计=100（一级/二级/三级各自）
  - 父子一致
  - 预警规则（产出+效益<60）

### Step5 生成评分标准
- 列表：按三级指标逐条生成
- 每条：
  - 左侧显示：指标名/解释/分值
  - 右侧生成：评分标准文本
  - 操作：采用/修改/重试/多模型对比

### Step6 完整指标体系（表格化）+ 问卷（可选）
- 指标体系表格：
  - 行：一级/二级/三级
  - 列：分值/解释/评分标准
- 拖拽排序：
  - 一级可上下拖拽，带动子级
  - 二级可上下拖拽，带动三级
  - 三级仅在本二级内重排
- 问卷生成：选择“需要/不需要”；生成后可编辑

### Step7 指标分析 + 得分表
- 输入表单：按评分标准要点生成多个输入框
- 每个要点字段：得分、原因（大白话）、扣分原因
- 输出：
  - 分析文本（按一级：决策/过程/产出/效益）
  - 自动生成得分表（含二级小计）

### Step8 经验做法
- 输入：可选上传工作总结/补充说明
- 输出：经验做法段落（可对比/可编辑）

### Step9 问题与原因
- 扣分点提取：从 Step7 得分表中筛选扣分项
- 选择器：勾选哪些纳入“问题清单”
- 风格：中立/尖锐/委婉

### Step10 建议
- 基于问题逐条生成建议
- 风格：中立/尖锐/委婉
- 逻辑校对：对“指标-分析-问题-建议”一致性给出提示

### Step11 综合评价分析与结论
- 自动汇总得分与等级
- 可视化：柱状图/表格（一级指标贡献度）
- 输出：整体概况、亮点、问题脉络、结论

### Step12 基本情况
- 模块化编辑：背景/内容/组织/资金/目标…（可增模块）

### Step13 工作开展情况
- 模块化编辑：目的/对象/范围/原则/指标/方法/标准/过程
- 顺序可调整（客户端可调顺序但不可改名称）

### Step14 生成评价报告（拼装/预览/全文编辑/导出）
- 章节编排：默认顺序 + 自定义拖拽排序
- 全文预览：分页/目录
- 编辑：段落级编辑（Tiptap）
- 导出：Word/PDF

## 4. 关键 UI 组件清单（可复用组件库）

### 4.1 通用基础组件
- `AppLayout`（顶栏 + 侧栏 + 内容区）
- `PageHeader`（标题 + 面包屑 + 操作区）
- `DataTable`（列表/筛选/分页）
- `ConfirmDialog` / `Toast` / `ErrorAlert`

### 4.2 工作流组件（核心）
- `StepSidebar`：14 步导航、完成状态、当前步高亮
- `GenerationPanel`：
  - 模型选择（单选/多选对比）
  - 温度档位：发散/普通/严谨
  - 生成按钮/停止/重试
  - 流式输出窗口
- `ModelCompareTabs`：多模型输出 Tab/分栏对比 + “设为最终版本”
- `ArtifactEditor`：成品编辑器（Markdown/富文本/结构化表格）
- `HistoryDrawer`：历史版本列表、差异对比、恢复为当前
- `TaskProgress`：队列状态（排队/运行/完成/失败/取消）+ 进度

### 4.3 文件与引用组件
- `FileUploader`：多文件、进度、失败重试、分组上传
- `FileList`：文件列表与预览入口
- `CitationCard`：引用/溯源展示（来源文件、页码、片段）

### 4.4 指标体系与表格组件
- `IndicatorTreeTable`：树表联动（一级/二级/三级）
- `ScoreValidationPanel`：分值校验结果与一键定位
- `DragSortTable`：拖拽排序（带层级约束）

### 4.5 导出与排版组件
- `ExportPanel`：格式（docx/pdf）、版式（经典/自定义）、字体/字号/行距
- `ExportJobsList`：导出任务列表与下载

## 5. 交互规范（必须统一的 UX 规则）

1) **主界面只显示最终版本**：
- 当前步骤的“成品区”永远只展示最终版本；
- 历史输出必须在“历史区/历史抽屉”查看，避免污染上下文。

2) **多模型对比统一体验**：
- 选择多模型 → 并行生成 → 输出按模型分 Tab
- 明确按钮：`设为最终版本`、`复制到成品`、`继续对话优化`

3) **长任务统一状态机**：
- queued/running/succeeded/failed/canceled
- 失败必须提供：错误摘要 + 重试按钮

4) **温度/Top-p 三档**：
- 发散 / 普通 / 严谨：前端只展示档位，不暴露连续滑杆

5) **权限与可见性**：
- 用户端：仅能访问自己项目
- 管理端：跨用户访问需二次确认（下载/删除/重置密码）

6) **90 天保留策略**：
- 前端提示“过期时间”
- 过期前可提醒下载

## 8. UI 风格与视觉建议（符合“科技感 + 暖色调”）

- 色彩：深色底 + 暖色强调（橙/琥珀）+ 高对比文字
- 信息层级：大量表格/文本时，必须用卡片分组与折叠面板减少视觉负担
- 表格密度：指标体系表格提供“紧凑/舒适”两档
