- **API接口规范文档**：每次添加/修改接口时更新
- **权限鉴权规范文档**：每次更改权限规则时更新
- **数据库表结构说明**：每次修改 schema 时更新
- **LangGraph工作流详细设计文档**：每次添加/修改 Step 时更新
- **前端Pinia状态管理设计文档**：每次添加/修改 store 时更新

### 如何贡献文档

1. 修改对应的 `.md` 文件
2. 提交 PR
3. 审查无误后 merge

### 文档规范

- 使用 Markdown 格式
- 代码示例必须包含注释和错误处理
- 包含表格时使用 GitHub Markdown 表格格式
- 长文档使用目录导航
- 包含版本号和最后更新时间
