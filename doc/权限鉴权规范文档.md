## 角色定义

系统中定义两个核心角色：

### 1. User（普通用户）

**特点**：
- 每个新注册用户的默认角色
- 拥有自己的项目、文件、对话等个人资源
- 可以创建项目、上传文件、执行工作流步骤
- **不能**访问管理端或他人资源

**权限范围**：
- 自己创建的项目：完全权限（读、写、删、执行）
- 他人的项目：无权限
- 系统配置：只读（模型列表、Prompt 概览，不含正文）

**应用场景**：
- 绩效评价员、部门负责人、HR

---

### 2. Admin（管理员）

**特点**：
- 由超级管理员或系统管理员赋予
- 拥有整个系统的管理权限
- 可以管理用户、配置 Prompt、维护知识库、查看审计日志

**权限范围**：
- 所有用户的项目：只读或审计权限
- 系统配置（Prompt、知识库、模型）：完全权限
- 用户管理：创建、禁用、查看使用情况
- 审计日志：完全读取权限

**应用场景**：
- 系统管理员、产品运营、内容管理

## 权限模型

### 基于角色的访问控制 (RBAC)

```
用户 (User)
  ├─ Role: user | admin
  ├─ Status: active | disabled
  └─ Projects (拥有或可见)

项目 (Project)
  ├─ Owner: User ID
  ├─ Files (上传的资料)
  ├─ Step Outputs (各步骤产出)
  └─ Conversations (与 AI 的对话)

管理资源
  ├─ Prompts (按 step_code 组织)
  ├─ Knowledge Bases (按 step_code 组织)
  └─ Model Configurations
```

### 权限粒度

| 资源 | 操作 | 粒度 | 判断逻辑 |
|-----|------|------|---------|
| Project | 创建 | 全局 | 已登录 user |
| Project | 读取 | 对象 | 项目所有者 \| admin |
| Project | 更新 | 对象 | 项目所有者 \| admin |
| Project | 删除 | 对象 | 项目所有者 \| admin |
| File | 上传 | 对象 | 该项目所有者 \| admin |
| File | 读取 | 对象 | 该项目所有者 \| admin |
| Step | 执行 | 对象 | 该项目所有者 \| admin |
| Prompt | 读取 | 权限受限 | user: 仅知是否有配置；admin: 查看正文 |
| Prompt | 修改 | 全局 | admin 仅 |
| Model | 管理 | 全局 | admin 仅 |
| Audit Log | 读取 | 对象 | user: 自己的操作；admin: 全部 |
| User | 管理 | 全局 | admin 仅 |

## 认证与授权流程

### 认证流程 (Authentication)

```
┌─────────────────────────────┐
│  用户在前端输入用户名+密码   │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  POST /api/v1/auth/login        │
│  {username, password}           │
└──────────────┬──────────────────┘
               │
               ▼
┌──────────────────────────────┐
│ Backend: AuthService         │
│  1. 查询 users 表获取密码哈希  │
│  2. bcrypt 校验密码           │
│  3. 生成 Token: "user:<uuid>" │
└──────────────┬────────────────┘
               │
               ▼
┌──────────────────────────────┐
│  Response: {                 │
│    access_token: "user:xxx", │
│    token_type: "bearer"      │
│  }                           │
└──────────────┬────────────────┘
               │
               ▼
┌──────────────────────────────┐
│ 前端: 将 token 存入           │
│ localStorage['ef_token']      │
└──────────────────────────────┘
```

**Token 格式**：
- 格式：`user:<user_id>`（user 是固定前缀）
- 存储位置：`localStorage['ef_token']`
- 传输方式：HTTP Header `Authorization: Bearer user:<user_id>`
- **注意**：当前实现中 Token 不含过期时间，生产环境需补充 JWT 的 exp 字段

### 授权流程 (Authorization)

对每个受保护的接口，执行以下授权检查：

```
┌────────────────────────────────┐
│ 请求进入 FastAPI               │
│ Header: Authorization: Bearer.. │
└──────────────┬─────────────────┘
               │
               ▼
┌────────────────────────────────┐
│ Dependency: get_actor_user_id  │
│  1. 解析 Bearer token           │
│  2. 提取 user_id               │
│  3. 查询 users 表验证用户存在    │
└──────────────┬─────────────────┘
               │
               ▼
        ┌──────┴──────┐
        │             │
        ▼             ▼
   认证成功        Token 无效
   │              │
   │              └─► 401 Unauthorized
   │
   ▼
┌────────────────────────────────┐
│ 检查角色权限                    │
│ 根据接口类型判断是否需要 admin  │
└──────────────┬─────────────────┘
               │
        ┌──────┴──────┐
        │             │
        ▼             ▼
   权限检查         权限不足
   通过            │
   │              └─► 403 Forbidden
   │
   ▼
┌────────────────────────────────┐
│ 检查资源所有权                  │
│ 例：该项目是否属于当前用户？     │
└──────────────┬─────────────────┘
               │
        ┌──────┴──────┐
        │             │
        ▼             ▼
   所有权          所有权
   检查通过        检查失败
   │              │
   │              └─► 403 Forbidden
   │
   ▼
┌────────────────────────────────┐
│ 执行业务逻辑                    │
└────────────────────────────────┘
```

## 资源所有权判断

### 规则 1：项目所有权

```sql
SELECT * FROM projects WHERE id = ? AND (user_id = ? OR ? = 'admin')
```

**伪代码**：
```python
def check_project_access(project_id: str, user_id: str, user_role: str) -> bool:
    project = db.query(Project).filter(Project.id == project_id).first()
    if not project:
        return False  # 404

    if user_role == 'admin':
        return True   # admin 可访问任意项目

    if project.user_id == user_id:
        return True   # 项目所有者

    return False      # 403 Forbidden
```

### 规则 2：文件所有权

文件必须属于某个项目，检查顺序：

1. 验证项目所有权（参考规则 1）
2. 验证文件属于该项目：`file.project_id == project_id`

### 规则 3：步骤操作权限

- 步骤生成、查询、保存都依赖项目所有权
- 如果用户无法访问项目，则无法操作任何步骤

### 规则 4：对话权限

- 对话必须属于某个项目
- 检查项目所有权即可

## 鉴权实现细节

### 后端实现

#### 1. 提取用户身份（Dependency Injection）

**文件**：`Backend/app/api/deps.py`

```python
from fastapi import Header, Depends
from sqlalchemy.orm import Session
from app.db.session import get_db
from app.core.errors import unauthorized

def get_actor_user_id(
    authorization: str | None = Header(default=None),
    db: Session = Depends(get_db)
) -> str:
    """
    从 Authorization header 提取用户 ID。
    格式：Authorization: Bearer user:<user_id>
    """
    if not authorization or not authorization.startswith("Bearer "):
        raise unauthorized("未提供有效的 Authorization header")

    token = authorization.removeprefix("Bearer ").strip()
    if not token.startswith("user:"):
        raise unauthorized("Token 格式无效")

    user_id = token.split(":", 1)[1]

    # 验证用户存在
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise unauthorized("用户不存在")

    if user.status == "disabled":
        raise unauthorized("用户已被禁用")

    return user_id
```

#### 2. 检查角色权限

```python
from app.core.errors import forbidden

def require_admin(
    actor_user_id: str = Depends(get_actor_user_id),
    db: Session = Depends(get_db)
) -> User:
    """
    检查用户是否为 admin。
    """
    user = db.query(User).filter(User.id == actor_user_id).first()
    if user.role != "admin":
        raise forbidden("需要管理员权限")
    return user
```

#### 3. 检查资源所有权

```python
def check_project_access(
    project_id: str,
    actor_user_id: str,
    db: Session,
    allow_admin: bool = True
) -> Project:
    """
    检查用户是否可以访问项目。
    如果 allow_admin=True，admin 总是可以访问；
    否则只有项目所有者可以访问。
    """
    project = db.query(Project).filter(Project.id == project_id).first()
    if not project:
        raise not_found("项目不存在")

    user = db.query(User).filter(User.id == actor_user_id).first()

    if allow_admin and user.role == "admin":
        return project

    if project.user_id == actor_user_id:
        return project

    raise forbidden("您无权访问该项目")
```

#### 4. 在路由中使用

```python
from fastapi import APIRouter, Depends, Body

router = APIRouter()

@router.get("/{project_id}")
def get_project(
    project_id: str,
    actor_user_id: str = Depends(get_actor_user_id),
    db: Session = Depends(get_db)
):
    """获取项目（需要所有权检查）"""
    project = check_project_access(project_id, actor_user_id, db)
    return project


@router.post("/prompts")
def create_prompt(
    payload: PromptCreateRequest,
    user: User = Depends(require_admin),  # 直接要求 admin
    db: Session = Depends(get_db)
):
    """创建 Prompt（仅管理员）"""
    return service.create_prompt(payload.step_code, payload.title, payload.content)
```

### 前端实现

#### 1. 路由守卫

**文件**：`frontend/src/router/index.ts`

```typescript
import { useAuthStore } from '@/stores/auth'

const router = createRouter({
  // ...routes
})

router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()

  // 需要认证的页面
  if (to.meta.requiresAuth && !authStore.isLoggedIn) {
    next('/login')
    return
  }

  // 需要 admin 的页面
  if (to.meta.requiresAdmin && authStore.user?.role !== 'admin') {
    next('/forbidden')
    return
  }

  next()
})
```

#### 2. 页面元数据


#### 4. API 请求拦截

**文件**：`frontend/src/services/api.ts`

```typescript
import axios from 'axios'
import { useAuthStore } from '@/stores/auth'

const api = axios.create({
  baseURL: '/api/v1'
})

// 请求拦截器：自动添加 token
api.interceptors.request.use((config) => {
  const authStore = useAuthStore()
  if (authStore.token) {
    config.headers.Authorization = `Bearer ${authStore.token}`
  }
  return config
})

// 响应拦截器：处理 401/403
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Token 过期或无效，登出
      useAuthStore().logout()
      window.location.href = '/login'
    } else if (error.response?.status === 403) {
      // 权限不足，显示错误提示
      ElMessage.error('您没有权限执行此操作')
    }
    return Promise.reject(error)
  }
)

export default api
```

## 常见场景

### 场景 1：用户 A 尝试访问用户 B 的项目

**请求**：`GET /api/v1/projects/proj-b-uuid`  
**用户**：user-a-uuid

**鉴权流程**：
1. ✅ 认证成功（token 有效）
2. ✅ 角色检查通过（是 user）
3. ❌ 资源检查失败（项目所有者是 user-b）
4. **结果**：403 Forbidden

**响应**：
```json
{
  "detail": "您无权访问该项目"
}
```

---

### 场景 2：管理员访问任意项目

**请求**：`GET /api/v1/projects/proj-b-uuid`  
**用户**：admin-uuid

**鉴权流程**：
1. ✅ 认证成功
2. ✅ 角色检查通过（是 admin）
3. ✅ 资源检查通过（admin 可访问任意资源）
4. **结果**：200 OK

**响应**：返回项目详情

---

### 场景 3：普通用户尝试创建 Prompt

**请求**：`POST /api/v1/admin/prompts`  
**用户**：user-uuid

**鉴权流程**：
1. ✅ 认证成功
2. ❌ 角色检查失败（需要 admin，但是 user）
3. **结果**：403 Forbidden

**响应**：
```json
{
  "detail": "需要管理员权限"
}
```

---

### 场景 4：Admin 查看他人项目的 Step 输出

**请求**：`GET /api/v1/steps/step1/result?project_id=proj-b-uuid`  
**用户**：admin-uuid

**鉴权流程**：
1. ✅ 认证成功
2. ✅ 角色检查通过（是 admin）
3. ✅ 资源检查通过（admin 可访问任意项目的步骤）
4. **结果**：200 OK

**响应**：返回步骤结果

---

### 场景 5：Token 过期/无效

**请求**：任何受保护端点  
**Header**：`Authorization: Bearer user:invalid-uuid`

**鉴权流程**：
1. ❌ 认证失败（用户不存在或已禁用）
2. **结果**：401 Unauthorized

**响应**：
```json
{
  "detail": "用户不存在"
}
```

## 安全建议

### 1. Token 安全

- ✅ 使用 HTTPS 传输 token
- ✅ Token 存储在 `localStorage` 时应标记为 `HttpOnly`（需要 cookies 实现）
- ✅ 前后端都需验证 token 有效性
- ❌ 避免在 URL 中传递 token
- ❌ 避免在错误消息中暴露 token

**改进方案**（生产环境）：

```python
# 使用 PyJWT 生成带过期时间的 JWT token
import jwt
from datetime import datetime, timedelta

ALGORITHM = "HS256"
SECRET_KEY = "your-secret-key"
TOKEN_EXPIRE_MINUTES = 24 * 60  # 24 小时

def create_token(user_id: str) -> str:
    payload = {
        "sub": user_id,
        "iat": datetime.utcnow(),
        "exp": datetime.utcnow() + timedelta(minutes=TOKEN_EXPIRE_MINUTES)
    }
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def verify_token(token: str) -> str:
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id: str = payload.get("sub")
        if user_id is None:
            raise unauthorized("Invalid token")
        return user_id
    except jwt.ExpiredSignatureError:
        raise unauthorized("Token expired")
    except jwt.InvalidTokenError:
        raise unauthorized("Invalid token")
```

### 2. 密码安全

- ✅ 使用 bcrypt 等强哈希算法存储密码
- ✅ 密码长度要求至少 8 位，包含大小写和数字
- ✅ 实现密码修改和重置流程
- ❌ 不要在日志中记录密码明文
- ❌ 不要在错误信息中暴露用户是否存在

**示例**：
```python
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain_password: str, hashed: str) -> bool:
    return pwd_context.verify(plain_password, hashed)

# 登录时
if not verify_password(payload.password, user.password_hash):
    raise unauthorized("用户名或密码错误")  # 故意不区分
```

### 5. 审计日志

**必须记录的操作**：
- 登录/登出
- 项目创建/删除
- Prompt/知识库修改
- 模型配置变更
- 用户禁用
- 敏感数据访问

**示例实现**：
```python
def audit_log(
    actor_user_id: str,
    action: str,  # "create", "update", "delete", "publish"
    target_type: str,  # "project", "prompt", "model", "user"
    target_id: str,
    before_data: dict = None,
    after_data: dict = None,
    db: Session = None
):
    """
    记录审计日志。
    - action: 操作类型
    - target_type: 目标资源类型
    - target_id: 目标资源 ID
    - before_data: 修改前数据
    - after_data: 修改后数据
    """
    log_entry = AuditLog(
        actor_user_id=actor_user_id,
        action=action,
        target_type=target_type,
        target_id=target_id,
        before_data=json.dumps(before_data) if before_data else None,
        after_data=json.dumps(after_data) if after_data else None,
        created_at=datetime.utcnow()
    )
    db.add(log_entry)
    db.commit()
```

### 如何升级用户为管理员

**建议方式**：使用管理 CLI 或后台数据库操作，不通过 API

```python
# Backend/scripts/promote_admin.py
import sys
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from app.models.user import User

engine = create_engine("postgresql://...")
Session = sessionmaker(bind=engine)
session = Session()

username = sys.argv[1]  # 从命令行参数获取用户名
user = session.query(User).filter(User.username == username).first()
if user:
    user.role = "admin"
    session.commit()
    print(f"✅ 已将 {username} 升级为管理员")
else:
    print(f"❌ 用户 {username} 不存在")
```
