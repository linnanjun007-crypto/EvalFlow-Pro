# EvalFlow Pro 智能体开发计划书

## 1. 项目背景

EvalFlow Pro 的后端将基于 `LangGraph` 与 `LangChain` 构建智能体系统，服务于用户端 14 步评价工作流与管理端配置系统。

后端要解决的核心问题包括：
- 让每一步评价工作流可编排、可恢复、可回滚
- 支持多模型协作与输出对比
- 支持 Prompt、知识库、模型配置的版本化管理
- 支持历史、审计、用量、导出等能力
- 为后续产品化与规模化留出扩展空间

---

## 2. 建设目标

### 2.1 总目标
建立一个可维护、可扩展、可观测的智能体后端平台，为 EvalFlow Pro 的所有前端页面提供稳定服务。

### 2.2 分目标
- 构建统一的项目与工作流状态管理体系
- 通过 LangGraph 编排 Step1~Step14 的评价流程
- 通过 LangChain 封装模型调用、Prompt、工具链
- 支持历史版本管理、导出任务与审计日志
- 支持管理端对模型、Prompt、知识库的配置

---

## 3. 技术路线

### 3.1 核心框架
- **LangGraph**：负责工作流图谱、状态流转、分支、恢复、重试
- **LangChain**：负责模型接入、Prompt 组织、工具调用、输出解析

### 3.2 推荐后端栈
- **Python 3.11+**
- **FastAPI**：HTTP API 层
- **Pydantic**：请求与响应模型
- **SQLAlchemy / SQLModel**：数据持久化
- **PostgreSQL**：主数据库
- **Redis**：缓存、任务状态、短期会话
- **Celery / RQ / Dramatiq**：异步任务（可选）
- **MinIO / S3**：文件、导出包存储

### 3.3 模型接入建议
- OpenAI 系列
- Anthropic 系列
- 国内模型（Qwen、GLM 等）
- 通过模型注册表统一配置与切换

## 4. 系统架构

建议采用五层结构：

### 4.1 接口层
提供登录、项目、工作流、历史、导出、管理端配置等 API。

### 4.2 服务层
负责业务规则、权限控制、任务调度、版本管理与审计写入。

### 4.3 智能体编排层
使用 LangGraph 编排 14 步工作流，支持：
- 顺序执行
- 条件分支
- 局部重试
- 回退
- 恢复
- 并行对比

### 4.4 模型与工具层
使用 LangChain 封装：
- Prompt 模板
- LLM 调用
- 工具调用
- 输出结构化解析

### 4.5 存储层
保存：
- 用户
- 项目
- 步骤输出
- 历史版本
- 模型配置
- Prompt
- 知识库
- 审计日志
- 导出任务

## 6. LangGraph 工作流设计

### 6.1 工作流节点
建议主图包含以下节点：
- `start`
- `step1_collect_materials`
- `step2_extract_valid_docs`
- `step3_generate_indicator_tree`
- `step4_allocate_scores`
- `step5_generate_scoring_rules`
- `step6_build_full_table`
- `step7_analyze_scores`
- `step8_generate_best_practices`
- `step9_identify_problems`
- `step10_generate_suggestions`
- `step11_generate_summary`
- `step12_generate_basic_info`
- `step13_generate_work_progress`
- `step14_assemble_report`
- `export`
- `end`

### 6.2 节点职责

#### Step1 资料清单
- 读取上传文件列表
- 记录文件类型、名称、页数、来源
- 为后续解析建立索引

#### Step2 有效项目资料
- 提取有效文本
- 分类材料
- 建立引用来源
- 输出可编辑内容

#### Step3 指标体系
- 生成树形结构
- 支持一级 / 二级 / 三级指标
- 允许自定义模板

#### Step4 分值
- 自动分配分值
- 校验总分和父子层级
- 支持人工修正

#### Step5 评分标准
- 为三级指标生成评分标准
- 支持多模型输出对比

#### Step6 完整指标体系
- 汇总指标树、分值与评分标准
- 输出结构化表格

#### Step7 指标分析
- 根据评分标准要点分析得分
- 生成扣分原因

#### Step8 经验做法
- 从总结材料中提炼经验做法

#### Step9 问题与原因
- 从扣分项中提取问题
- 生成原因分析

#### Step10 建议
- 基于问题生成建议
- 保持与前文逻辑一致

#### Step11 综合评价
- 输出整体结论、等级与亮点

#### Step12 基本情况
- 生成背景、概况、组织等内容

#### Step13 工作开展情况
- 生成工作过程、范围、方法、原则等内容

#### Step14 评价报告
- 拼装全文
- 生成报告结构
- 提供导出前预览

## 14. 风险与注意事项

1. **Prompt 波动大**
- 需要版本化与回滚机制

2. **多模型输出差异大**
- 需要统一结构化输出

3. **长文档处理成本高**
- 需要分段、摘要与缓存策略

4. **工作流状态复杂**
- 需要清晰的状态机设计

5. **导出一致性要求高**
- 需要最终校对与锁定机制

---

## 15. 下一步建议

如果你准备从 `LangGraph + LangChain` 开始，建议第一批先做：

1. 后端工程初始化
2. 数据表设计
3. LangGraph 主图骨架
4. Step2 / Step3 / Step14 打通
5. 再逐步扩展到其余步骤

---

## 16. 总结

这份计划书的核心原则是：
- 先搭工作流状态机
- 再搭 Prompt / 知识库 / 模型体系
- 最后完善导出、审计、统计、版本管理

这样可以让 EvalFlow Pro 的智能体后端既能快速上线，又能持续演进。
